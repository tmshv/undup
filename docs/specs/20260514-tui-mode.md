# TUI mode for undup

## Goal

Add an interactive terminal UI to `undup` that lets the user browse detector findings, see total reclaimable bytes, multi-select members, and apply destructive actions (delete or move to a folder) without leaving the terminal. The TUI is built with [bubbletea](https://github.com/charmbracelet/bubbletea) and its companions (`bubbles`, `lipgloss`).

The TUI is a presentation layer over the existing detectors. It does not change detection logic.

## Invocation

A new top-level flag is added to `cmd/undup/main.go`:

```
undup [--tui|-i] [-m archives|hashsum|all] [-j N] <root>
```

- `--tui` / `-i`: when present, launches the TUI after wiring the scan pipeline. When absent, the current plain-text output behavior is preserved exactly as it is today.
- `--tui` is **orthogonal** to `-m`. `-m`'s existing default (`archives`) and existing semantics are unchanged; the TUI simply shows whichever detector(s) `-m` selects. To see both kinds in the TUI, pass `-m all` explicitly. The TUI's internal model handles a single-source feed transparently — the unused channel is `nil` and the corresponding `recv` command is never scheduled.
- `--tui` and a non-TTY stdout are not auto-detected; the user opts in explicitly.

## Scope

### In scope

- Unified view of archive findings and duplicate groups.
- Live streaming of findings into the table during scan.
- Per-member selection with collapsible groups.
- Standing status line showing scan state, selected count, and total reclaimable bytes.
- Two destructive actions: `delete` (irreversible) and `move-to-folder` (preserves path relative to scan root).
- Modal confirmation for both actions, with explicit recursive-directory warning when applicable.
- Inline text prompt for the move-target path.

### Out of scope (v1)

- Move to system trash (no library dependency, no `osascript` shellout). Deferred.
- Undo of completed actions.
- In-TUI filter or search.
- Re-scan after actions.
- Errors overlay listing per-member failure reasons (only an aggregate failure count is shown).
- Golden-frame snapshot tests for the rendered TUI.

## Architecture

```
cmd/undup/main.go
  ├── existing: runArchives, runDuplicates, runAll (untouched)
  └── new:      runTUI(root, workers, mode)

internal/scan/...   (unchanged)
  └── Walk, ArchiveDetector, DuplicateDetector

internal/tui/       (new package)
  ├── model.go      Model: findings, cursor, selection, mode, channels, status
  ├── finding.go    Unified Finding/Member types + adapters from scan types
  ├── update.go     Update(): keys, scan messages, action messages
  ├── view.go       View(): table, status line, modal, prompt
  ├── action.go     DeleteAction / MoveAction with rel-path layout
  ├── keys.go       Keybinding definitions
  ├── finding_test.go
  ├── action_test.go
  └── update_test.go
```

The bubbletea program owns the table state. Scan pipeline setup (Walk + tee + detectors) lives in `cmd/undup/main.go` exactly as it does today for `runAll`; the only difference is that the detector output channels are handed to `tui.Run(ctx, archCh, dupCh, scanRoot)` instead of being printed.

### Flow

1. `main.go` parses CLI. If `--tui` is set, validate `-j` and dispatch to `runTUI`.
2. `runTUI` constructs the scan pipeline based on `-m`:
    - `-m archives`: only run `ArchiveDetector` on the walk channel.
    - `-m hashsum`: only run `DuplicateDetector`.
    - `-m all`: run both via the existing `tee` helper. The tee continues to centralize walk-error logging.
3. `runTUI` calls `tui.Run(ctx, archCh, dupCh, scanRoot, mode)`. Channels that are unused for the current mode are passed as `nil`; the model treats `nil` channels as "already-done".
4. `tui.Run` constructs a `tea.Program` and runs it. On exit, returns to `main`, which exits 0.

### Concurrency model

- Two `tea.Cmd` functions block on `archCh` and `dupCh` respectively. Each reads one item, returns a typed `tea.Msg`, and is re-scheduled by `Update` until the channel closes. When a channel closes, a one-shot `scanHalfDoneMsg{source}` is returned and that side is marked done. The model's "scanning" status flips to "done" only when both sides are done.
- Filesystem actions (delete, move) run inside a `tea.Cmd`. They do their work synchronously inside the goroutine bubbletea spawns for the command, and return a single `actionResultMsg` when done. The UI cannot interleave a second action while one is in flight (the action keys are no-ops while `model.mode == modeApplying`).
- Directory-size resolution for archive findings runs in its own `tea.Cmd` triggered on first expansion of an archive group whose dir size is unresolved.

## Data model

```go
type Source int
const (
    SourceArchive Source = iota
    SourceDuplicate
)

type Finding struct {
    Source   Source
    Label    string   // "foo.zip" basename, or short SHA prefix "bea0aa1c"
    Members  []Member
    Expanded bool
}

type Member struct {
    Path     string
    Size     int64    // -1 if unresolved (archive directory pending walk)
    IsDir    bool
    Selected bool
}
```

### Adapters

- **`scan.ArchiveFinding{ArchivePath, DirPath, Size}` → `Finding`:**
    - `Members[0] = {Path: ArchivePath, Size: Size, IsDir: false, Selected: true}`
    - `Members[1] = {Path: DirPath, Size: -1, IsDir: true, Selected: false}`
    - `Label = filepath.Base(ArchivePath)`
- **`scan.DuplicateGroup{Paths, Size, SHA256}` → `Finding`:**
    - One `Member` per path; all share `Size`; `IsDir = false` (detector skips dirs).
    - `Selected = true` for all members except `Paths[0]` (the lexicographically-first path, which the detector already sorts).
    - `Label = hex.EncodeToString(SHA256[:4])`

### Directory-size resolution

When an archive group is expanded and its directory member has `Size == -1`, schedule a `tea.Cmd` that walks the directory accumulating regular-file sizes (`filepath.WalkDir` + `os.Lstat`). The result is delivered as `dirSizeMsg{findingIdx, memberIdx, size, err}`. On error, the size cell renders as `?` and that member becomes non-selectable (the reclaim total cannot account for it). The walk uses one goroutine; the user can move the cursor and toggle other groups while it's running.

## UI

ASCII layout:

```
┌─ undup ────────────────────────────────────────────────────────────┐
│ Scanning…   Selected: 4 / 27   Reclaim: 12.3 GiB                   │
├────────────────────────────────────────────────────────────────────┤
│ ▶ arc  foo.zip                                       12.0 MiB   2  │
│ ▼ arc  release-bundle.tar.gz                          1.2 GiB   2  │
│     [x] /scans/release-bundle.tar.gz                  1.2 GiB      │
│     [ ] /scans/release-bundle/                          …    (dir) │
│ ▶ dup  bea0aa1c  3 copies                             6.2 GiB   3  │
│ ▼ dup  4f17c2e0  4 copies                             9.0 GiB   4  │
│     [ ] /a/photos/IMG_0001.RAW                        3.0 GiB      │
│     [x] /b/photos/IMG_0001.RAW                        3.0 GiB      │
│     [x] /c/imports/IMG_0001.RAW                       3.0 GiB      │
│     [x] /d/copy/IMG_0001.RAW                          3.0 GiB      │
├────────────────────────────────────────────────────────────────────┤
│ ↑/↓ move  space toggle  ⏎ expand  a sel-group  d delete  m move  ? │
└────────────────────────────────────────────────────────────────────┘
```

### Columns

| Column     | Group row                           | Member row                         |
| ---------- | ----------------------------------- | ---------------------------------- |
| Indicator  | `▶` collapsed / `▼` expanded        | `[ ]` / `[x]` selection            |
| Kind       | `arc` or `dup`                      | (blank)                            |
| Label/Path | Finding label                       | Member path (left-truncated)       |
| Size       | Sum of selected member sizes        | Member size (or `…` while pending) |
| Count      | Number of members                   | `(dir)` annotation when applicable |

Paths longer than the available width are truncated from the left with `…`, preserving the basename.

### Status line

`Scanning…  Selected: N / X  Reclaim: H` where:
- `Scanning…` becomes `Done` once every active scan channel is closed (one or two channels depending on `-m`).
- `N` is the count of currently-selected members across all findings.
- `X` is the count of **selectable** members across all findings — members whose directory-size walk failed are excluded.
- `H` is the human-readable sum of selected member sizes (the existing `humanSize` helper is reused).

Transient toast lines (`Deleted N · failed M`, `Moved N to /path · failed M`, error messages) replace the standing status line for ~3 seconds and then revert.

### Keybindings

| Key                | Action                                            |
| ------------------ | ------------------------------------------------- |
| `↑` / `k`          | Move cursor up                                    |
| `↓` / `j`          | Move cursor down                                  |
| `g`                | Jump to first visible row                         |
| `G`                | Jump to last visible row                          |
| `Space`            | Toggle selection on current member                |
| `Enter`            | Expand / collapse current group                   |
| `a`                | Select all members of current group               |
| `A`                | Apply default selection rules to all groups       |
| `c`                | Clear all selections                              |
| `d`                | Delete selected → confirm modal                   |
| `m`                | Move selected → target prompt → confirm modal     |
| `?`                | Toggle help overlay                               |
| `q` / `Ctrl+C`     | Quit (closes any open modal first)                |

While the move prompt or the confirmation modal is open, all keys other than those listed for that modal are ignored.

### Move-target prompt

When `m` is pressed with at least one member selected:

```
│ Move 4 items (12.0 GiB) to: /tmp/undup-quarantine_         │
│ Enter to confirm · Esc to cancel                           │
```

- Pre-filled with `<cwd>/undup-quarantine` on first use.
- Subsequent uses in the same session pre-fill with the previous value.
- Implemented with `bubbles/textinput`.
- `Enter` → resolves the target path (`filepath.Abs`), then opens the confirmation modal.
- `Esc` → cancels, returns to the table without changing selection.

### Confirmation modal

```
┌─ confirm delete ──────────────────────────────────────────┐
│  Delete 4 files / directories?                            │
│  Total reclaim: 12.0 GiB                                  │
│                                                           │
│  Includes 1 directory (recursive).                        │
│                                                           │
│  [y] yes   [n / Esc] cancel                               │
└───────────────────────────────────────────────────────────┘
```

- The "Includes N directory (recursive)" line is shown only when at least one selected member has `IsDir = true`.
- Move modal substitutes `Delete` → `Move`, adds the resolved target path, and omits the recursive line (move is non-destructive in the same way as delete).
- `y` commits; `n`, `Esc`, or `q` cancels.

## Action layer

### `Action` interface

```go
type Action interface {
    Apply(member Member, scanRoot string) error
}

type DeleteAction struct{}
type MoveAction struct{ Target string; ScanRoot string }
```

The dispatcher (`applyActionCmd`) is a `tea.Cmd` that:

1. Collects all selected members across all findings.
2. **Deduplicates by absolute path** (a single file may legitimately appear in both an archive finding and a duplicate group; it must be acted on at most once).
3. For each unique member, calls `action.Apply`. Failures are collected, not fatal.
4. Returns `actionResultMsg{ok int; failed int; firstErr error}`.

### Delete

- Regular files: `os.Remove`.
- Directories (only reachable for `IsDir = true` archive-directory members): `os.RemoveAll`.

### Move

1. Reject the action up front if `filepath.Abs(target)` resolves to a path equal to `scanRoot` or contained within `scanRoot`. Emit a toast: `move target must be outside scan root`. No filesystem changes are made.
2. For each member:
    - Compute `rel, err := filepath.Rel(scanRoot, member.Path)`. If `err != nil` or `rel` begins with `..`, fall back to `filepath.Base(member.Path)` and surface as a failure to the result count (the file is not moved; the user picks a different target or a different selection).
    - `dest := filepath.Join(target, rel)`.
    - `os.MkdirAll(filepath.Dir(dest), 0o755)`.
    - Try `os.Rename(member.Path, dest)`. On `syscall.EXDEV` (cross-device), fall back to `copyFile(src, dest)` followed by `os.Remove(src)`. `copyFile` preserves mode bits via `os.Chmod`; it does not preserve ownership, xattrs, or symlinks (none of the detectors emit symlinks).
    - If any step fails before the source is removed, the source is left untouched.

### Post-action state update

- Members whose action succeeded are removed from their `Finding.Members`.
- `Finding`s left with fewer than 2 members are removed entirely from `model.findings` (a group of 1 has no reclaim value).
- The cursor is clamped to the new visible-row count.
- The toast `Deleted N · failed M` or `Moved N to /path · failed M` shows for ~3 seconds.

## Errors and edge cases

| Case                                              | v1 behavior                                                         |
| ------------------------------------------------- | ------------------------------------------------------------------- |
| Symlinks among duplicate paths                    | Detector skips them; none appear in the TUI.                        |
| Same file in both an archive finding and a dup group | Deduped by absolute path before action dispatch.                 |
| Selected member deleted out-of-band               | `os.Remove`/`os.Rename` errors; counted in `failed`.                |
| Move target equals source                         | Caught by the pre-check; reported as failure for that member.       |
| Move target inside scan root                      | Refused before any action runs; toast explains why.                 |
| Terminal resize                                   | `WindowSizeMsg` re-renders; columns re-truncate.                    |
| `Ctrl+C` mid-action                               | The in-flight `tea.Cmd` is awaited (no orphaned goroutines), then the program exits cleanly. |
| Empty result set after both channels close        | Status line shows `Done · no findings`; quit on `q` / `Ctrl+C`.     |
| Directory-size walk error for an archive member   | Member size renders as `?`; member becomes non-selectable.          |

Walk errors continue to be logged to stderr by the existing tee in `main.go`; they are not interleaved into the TUI.

## Testing strategy

- `internal/tui/finding_test.go` — table tests for the adapters. Cover archive→Finding, duplicate→Finding, default-selection rules, and label formatting.
- `internal/tui/action_test.go` — `t.TempDir()`-rooted tests for delete and move:
    - Delete file, delete directory (recursive).
    - Move with relative-path layout (`scanRoot/sub/file.bin` → `target/sub/file.bin`).
    - Move creates missing parent directories.
    - Move target inside scan root is refused.
    - Move handles the `rel = ".."` fallback case (member outside scan root).
    - Cross-device fallback path is covered by a small unit test that injects a faked `os.Rename` error or skipped if not feasible to force.
    - Dedup: a member listed in both an archive Finding and a duplicate Finding is acted on only once.
- `internal/tui/update_test.go` — drives `Update()` synthetically:
    - Key handling: cursor movement, toggle, expand/collapse, modal open/close, `a`/`A`/`c`.
    - Scan message handling: append to findings, mark half-done, mark fully-done.
    - Modal flow: `m` → prompt → `Enter` → confirm → action dispatch.
- No golden-frame `View()` tests.

## Acceptance criteria

- `undup --tui <root>` launches the TUI; without `--tui` the tool behaves byte-for-byte identically to the current `master`.
- The TUI shows findings from whichever detector(s) `-m` selects (`-m all` for both, `-m archives` for just archives, `-m hashsum` for just duplicates). `-m`'s existing default of `archives` applies if the user does not pass `-m`.
- The status line's reclaim figure updates in real time as the user toggles selections.
- A user can: select members, press `d`, confirm, and observe the affected files removed from the filesystem and from the table.
- A user can: select members, press `m`, enter a target outside the scan root, confirm, and observe the affected files relocated under the target, preserving their scan-root-relative paths.
- A move with target equal to or inside the scan root is rejected with a clear toast and no filesystem changes.
- A delete that includes any directory shows the recursive-directory warning in the confirmation modal.
- All new tests pass via `go test ./...`. `go vet ./...` and `go build ./...` succeed.
